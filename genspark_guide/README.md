## 젠스파크란?

### 한 줄 정의

**엑셀/CSV 데이터를 업로드하면 AI가 자동으로 전문 프레젠테이션(PPT)을 생성해주는 도구**

### 기존 작업 방식 vs 젠스파크

| 단계 | 기존 방식 (수작업) | 젠스파크 |
|------|-------------------|----------|
| 데이터 정리 | 2-3시간 | 자동 분석 (30초) |
| 슬라이드 구성 | 3-4시간 (레이아웃, 디자인) | AI 자동 생성 (2분) |
| 표/차트 생성 | 1-2시간 (수동 작업) | 자동 생성 |
| 디자인 일관성 | 수동 조정 필요 | 템플릿 기반 자동 유지 |
| **총 소요 시간** | **8-10시간** | **15분** |
| **효율** | - | **32배** ⚡ |

### 차별점

- **데이터 중심 접근**: 정보 입력이 아닌 데이터 업로드로 시작
- **템플릿 재사용**: 기존 템플릿 ID로 브랜딩 일관성 유지
- **자동 구조화**: 표지/목차/섹션/차트/표 자동 배치
- **즉시 내보내기**: PPTX/PDF 즉시 다운로드

## 핵심 기능

### 1. 데이터 자동 분석

**지원 형식:**
- Excel (.xlsx, .xls)
- CSV
- Google Sheets (연동)
- 구조화된 JSON

**자동 처리:**
- 헤더 인식 및 카테고리 분류
- 데이터 타입 자동 판단 (텍스트/숫자/날짜)
- 중요도 우선순위 분석
- 섹션별 그룹핑

### 2. 프레젠테이션 자동 생성

**자동 생성 페이지 유형:**
- 표지 (제목, 부제목, 날짜)
- 목차 (섹션 자동 구조화)
- 섹션 제목 페이지
- 글머리 기호 페이지 (요약 항목)
- 표 형식 페이지 (상세 데이터)
- 차트/그래프 (자동 시각화)
- 다이어그램 (프로세스, 구조)
- 타임라인/일정표
- 강조 페이지 (핵심 메시지)
- Q&A/마무리 페이지

### 3. 템플릿 시스템

**템플릿 ID 활용:**
```
기존 템플릿 ID: example-template-id-xxxxx
→ 브랜딩, 색상, 폰트, 레이아웃 자동 적용
```

**이점:**
- 회사 브랜딩 일관성 유지
- 반복 작업 효율화
- 템플릿 라이브러리 구축

### 4. 스마트 시각화

**자동 생성:**
- 데이터 → 막대/원형/꺾은선 차트
- 비교표 → 매트릭스 테이블
- 프로세스 → 플로우 다이어그램
- 일정 → 타임라인

**강조 기능:**
- 중요 항목 자동 하이라이트
- 컬러 코딩 (빨강/노랑/초록)
- 아이콘 자동 매칭

## 시작하기

### 1단계: 데이터 준비

**구조화된 엑셀 파일 예시:**

| 섹션 | 항목명 | 설명 | 상태 | 우선순위 |
|------|--------|------|------|----------|
| 준비사항 | 시스템 계정 | 활성화된 계정 필요 | 완료 | 높음 |
| 기술검증 | 접근권한 설정 | 권한 구성 | 진행중 | 높음 |
| 거버넌스 | 정책 문서화 | 규정 준수 | 대기 | 중간 |

**핵심 원칙:**
- 첫 행은 헤더로 명확히
- 카테고리/섹션 컬럼 포함
- 빈 셀 최소화
- 일관된 데이터 형식

### 2단계: 파일 업로드

1. https://www.genspark.ai/ 접속
2. "Upload Data" 또는 "Create Presentation" 클릭
3. 엑셀/CSV 파일 드래그 앤 드롭
4. (선택) 기존 템플릿 ID 입력

### 3단계: AI 생성 대기

**자동 처리 과정:**
- 데이터 구조 분석 (5초)
- 섹션 구성 설계 (10초)
- 페이지 생성 (30초~2분)
- 템플릿 적용 (즉시)

### 4단계: 결과 확인 & 수정

**View & Export 기능:**
- 온라인 프리뷰
- 페이지별 편집 가능
- PPTX/PDF 내보내기
- 템플릿 저장

## 기본 워크플로우

### 워크플로우 A: 신규 프레젠테이션 생성

```
1. 엑셀 데이터 준비 (구조화)
   ↓
2. Genspark 업로드
   ↓
3. 프롬프트 입력 (선택)
   "인증 준비 가이드를 전문 프레젠테이션으로 작성"
   ↓
4. 자동 생성 (2분)
   ↓
5. 온라인 프리뷰 확인
   ↓
6. 미세 조정 (페이지 추가/삭제/편집)
   ↓
7. PPTX 다운로드
```

### 워크플로우 B: 기존 템플릿 활용

```
1. 템플릿 ID 확인
   예: example-template-id-xxxxx
   ↓
2. 새 데이터 업로드
   ↓
3. 프롬프트에 템플릿 ID 포함
   "템플릿 ID example-template-id...를 활용하여 Q1 실적 보고서 생성"
   ↓
4. 자동 생성 (디자인 일관성 유지)
   ↓
5. 내보내기
```

## 실전 활용법

### 사례 1: 인증 준비 가이드

**시나리오:**
- ISO/규정 인증 준비 체크리스트 (20개 필수 항목 + 50개 상세 검증 항목)
- 전문 프레젠테이션으로 변환 필요

**실행 과정:**

1. **데이터 업로드**
   - Certification-Checklist.xlsx 업로드

2. **프롬프트 입력**
   ```
   기업 표준 템플릿 (ID: example-template-id-xxxxx)을
   활용하여 인증 준비 가이드 프레젠테이션 생성.

   포함 내용:
   - 필수 항목 20개
   - 6개 검증 영역 (문서/프로세스/기술/보안/운영/관리)
   - 준비 일정표
   ```

3. **자동 생성 결과 (16페이지)**
   - 표지
   - 목차 (주요 섹션)
   - 인증 프로그램 소개
   - 필수 항목 요약 (글머리 기호)
   - 검증 영역 다이어그램
   - 상세 검증 항목 (표 형식, 6페이지)
   - 준비 일정표 (타임라인, 중요 항목 강조)
   - 핵심 가치 강조
   - Q&A

4. **추가 요청 (참고 이미지 활용)**
   ```
   참고 이미지처럼 준비 일정표를 14페이지에 추가해줘.
   시작일자/완료일자/비고 포함, 상위 3개 항목 빨간 박스 강조.
   ```

5. **결과**
   - 일정표 자동 삽입
   - 시각적 강조 자동 적용

**소요 시간:** 15분 (수작업 대비 32배 효율)

### 사례 2: 제품 스펙 시트 → 영업 프레젠테이션

**엑셀 데이터:**

| 기능 | 사양 | 경쟁사 A | 경쟁사 B | 우위점 |
|------|------|----------|----------|--------|
| 처리속도 | 10ms | 25ms | 30ms | 2.5배 빠름 |
| 가격 | $99 | $149 | $129 | 33% 저렴 |

**프롬프트:**
```
제품 스펙 데이터를 영업팀용 프레젠테이션으로 변환.
- 경쟁 우위 시각화 (차트)
- 고객 가치 제안 강조
- 가격 비교표 포함
```

**자동 생성:**
- 경쟁사 비교 차트
- ROI 계산 페이지
- 고객 성공 사례 (데이터 기반)
- CTA (구매 유도) 마무리

### 사례 3: 컴플라이언스 체크리스트 → 감사 보고서

**엑셀 데이터:**

| 항목 | 요구사항 | 현재상태 | 담당자 | 마감일 |
|------|----------|----------|--------|--------|
| GDPR | 데이터 암호화 | 완료 | 김철수 | 2024-03-01 |
| ISO27001 | 접근통제 | 진행중 | 이영희 | 2024-03-15 |

**생성 결과:**
- 준수 현황 요약 (차트)
- 항목별 상세 (표)
- 리스크 분석 (빨강/노랑/초록 강조)
- 실행 계획

## 템플릿 시스템

### 템플릿 ID 관리

**템플릿 저장:**
1. 잘 나온 프레젠테이션 생성 후
2. "Save as Template" 클릭
3. 템플릿 ID 복사 (예: example-template-id-xxxxx)
4. 노션/문서에 기록

**템플릿 재사용:**
```
프롬프트에 포함:
"템플릿 ID [복사한 ID]를 활용하여 [새 주제] 프레젠테이션 생성"
```

### 템플릿 라이브러리 구축

**추천 분류:**

| 카테고리 | 템플릿명 | 템플릿 ID | 용도 |
|----------|----------|-----------|------|
| 비즈니스 | 분기 실적 보고서 | abc123... | 경영진 보고 |
| 기술 | 아키텍처 리뷰 | def456... | 기술 설계 검토 |
| 영업 | 제품 제안서 | ghi789... | 고객 발표 |
| 컴플라이언스 | 감사 체크리스트 | jkl012... | 내부 감사 |

**노션 템플릿 관리 예시:**
```
데이터베이스 구조:
- 템플릿명
- 템플릿 ID
- 카테고리
- 최종 수정일
- 사용 횟수
- 비고
```

## 고급 기능

### 1. 참고 이미지 활용

**사용법:**
```
프롬프트:
"첨부한 참고 이미지와 유사한 스타일로 일정표를 14페이지에 추가"
```

**효과:**
- 특정 디자인 패턴 재현
- 시각적 일관성 유지
- 복잡한 레이아웃 구현

### 2. 부분 수정 요청

**페이지 단위 수정:**
```
"5페이지의 표를 차트로 변경해줘"
"8페이지에 아이콘 추가"
"12페이지 제목을 'Summary'로 변경"
```

### 3. 스타일 지정

**프롬프트 예시:**
```
생성 시 다음 스타일 적용:
- 색상: 기업 CI (파랑 #0066CC, 회색 #555555)
- 폰트: 제목 Montserrat Bold, 본문 Open Sans
- 톤: 전문적이면서 간결하게
```

### 4. 데이터 강조 규칙

**자동 강조 설정:**
```
프롬프트:
"우선순위 '높음'은 빨간색, '중간'은 노란색, '낮음'은 초록색으로 표시"
"완료율 80% 이상은 굵은 글씨로 강조"
```

## 효율 극대화 전략

### 1. 데이터 표준화

**엑셀 템플릿 준비:**

```
공통 구조:
- A열: 섹션/카테고리
- B열: 항목명
- C열: 설명
- D열: 상태/값
- E열: 우선순위/비고
```

**이점:**
- 일관된 입력 → 예측 가능한 출력
- 템플릿 재사용 효율 향상
- 팀원 간 협업 용이

### 2. 프롬프트 라이브러리

**노션에 저장할 프롬프트 예시:**

**카테고리: 비즈니스 보고서**
```
템플릿 ID [template-id]를 활용하여 [분기] 실적 보고서 생성.
포함 내용:
- 매출/이익 차트
- 전분기 대비 증감률 표
- 주요 성과 3가지 강조
- 다음 분기 목표
```

**카테고리: 기술 문서**
```
[시스템명] 아키텍처 리뷰 프레젠테이션 생성.
포함 내용:
- 컴포넌트 다이어그램
- 기술 스택 표
- 성능 지표 차트
- 개선 로드맵
```

### 3. 체인 워크플로우

**단계별 정제:**

1. **1차 생성**: 데이터 업로드 + 기본 프롬프트
2. **2차 수정**: "5페이지 차트를 막대에서 원형으로 변경"
3. **3차 강화**: "참고 이미지와 같은 스타일로 일정표 추가"
4. **최종 확인**: PPTX 내보내기 → 파워포인트에서 미세 조정

### 4. 템플릿 버전 관리

**Git 스타일 관리:**

```
v1.0_기본_비즈니스템플릿 (ID: abc123...)
v1.1_비즈니스_색상수정 (ID: def456...)
v2.0_비즈니스_레이아웃변경 (ID: ghi789...)
```

**노션 변경 이력:**
- 수정일
- 변경 사항
- 템플릿 ID
- 사용 권장 여부

## 피해야 할 실수

### 1. 비구조화된 데이터 업로드

**❌ 나쁜 예:**
- 헤더 없는 엑셀
- 병합된 셀 과다 사용
- 일관성 없는 데이터 형식
- 빈 행/열 다수 포함

**✅ 좋은 예:**
- 명확한 헤더 (첫 행)
- 단일 셀 데이터
- 통일된 형식 (날짜, 숫자, 텍스트)
- 불필요한 빈 공간 제거

### 2. 프롬프트 너무 모호

**❌ 나쁜 예:**
```
"프레젠테이션 만들어줘"
```

**✅ 좋은 예:**
```
템플릿 ID example-template-id...를 활용하여 Q1 실적 보고서 생성.
포함:
- 매출 차트 (막대)
- 전년 대비 증감률 표
- 주요 성과 3가지 (강조)
- 다음 분기 목표 (타임라인)
```

### 3. 생성 결과 그대로 사용

**문제:**
- AI가 데이터 해석 오류 가능
- 시각적 조정 필요
- 비즈니스 맥락 누락 가능

**해결:**
- 반드시 프리뷰 확인
- 핵심 메시지 재검토
- 파워포인트에서 최종 편집

### 4. 템플릿 ID 미관리

**문제:**
- 템플릿 ID 분실
- 일관성 없는 디자인
- 재작업 필요

**해결:**
- 노션/문서에 템플릿 ID 기록
- 프로젝트별 템플릿 매핑
- 버전 관리

### 5. 참고 이미지 미활용

**놓치는 기회:**
- 특정 스타일 재현 불가
- 디자인 요구사항 전달 어려움

**활용 방법:**
- 원하는 디자인 스크린샷 첨부
- "이 이미지처럼..." 프롬프트
- 레이아웃 참고 이미지 제공
